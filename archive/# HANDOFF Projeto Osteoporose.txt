## CHECKLIST 2: DEV SENIOR FRONTEND

```
□ CASCATA
  - Ordem dos arquivos CSS no index.html?
  - Especificidade de cada regra?
  - Qual regra VENCE?

□ COMPUTED STYLE
  - Valor FINAL da propriedade?
  - DevTools confirma?

□ BOX MODEL
  - Height definida em toda cadeia (pai → filho)?
  - Elemento quebra fluxo? (position, float)

□ FLEX/GRID
  - Container tem display correto?
  - Filhos expandem? (flex:1)

□ INLINE VS EXTERNO
  - Inline sobrescrevendo?
  - Especificidade inline (1,0,0,0) vence tudo

□ ISOLAMENTO
  - Problema neste elemento ou herdado?
```

---

## CHECKLIST 3: ONDE COLOCAR O FIX

```
□ QUANDO CSS EXTERNO (viewer.css, base.css):
  - Padrão se repete em múltiplos elementos?
  - É comportamento padrão do sistema?
  - Regra já existe mas não funciona?
  → Corrigir no CSS global

□ QUANDO HTML:
  - Estrutura semântica errada?
  - Ordem dos elementos incorreta?
  - Classe faltando ou errada?
  → Ajustar estrutura

□ QUANDO INLINE:
  - Exceção ÚNICA que não se repete?
  - Override temporário para teste?
  → Último caso antes de polish

□ QUANDO POLISH.CSS:
  - Nenhuma opção acima resolve?
  - Precisa de classe utilitária nova?
  → ÚLTIMO RECURSO
```
